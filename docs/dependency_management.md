# Dependency Management

This project uses two complementary files for managing R package dependencies.
They serve different purposes and both should be kept up to date.

## `renv.lock` — Local Development

`renv.lock` is managed by the [`renv` package](https://rstudio.github.io/renv/) and ensures that all developers working on this project locally use the exact same package versions. It is the R equivalent of `package-lock.json` in Node.js projects.

When you clone this repo and want to get your local environment set up, install `renv` and then restore the project library:

```r
install.packages("renv")
renv::restore()
```

This will install all dependencies at the versions recorded in `renv.lock`.

### Day-to-day renv workflow

| Task | Command |
|---|---|
| Restore deps from lockfile | `renv::restore()` |
| Install a new package | `renv::install("packageName")` |
| Save current state to lockfile | `renv::snapshot()` |
| Check for inconsistencies | `renv::status()` |

After installing any new packages, always run `renv::snapshot()` and commit the updated `renv.lock` so other developers get the new dependency when they next run `renv::restore()`.

> **Note:** The `renv/library/` folder is gitignored by design — only `renv.lock` is committed, analogous to how `node_modules/` is gitignored and only the lockfile is committed in Node projects.

## `manifest.json` — Posit Connect Deployment

`manifest.json` is generated by the [`rsconnect` package](https://rstudio.github.io/rsconnect/) and is used by **Posit Connect** to rebuild the app's environment on the server when the app is deployed. It records package dependencies, the R version, and metadata that Connect needs to provision the runtime.

It is updated by running:

```r
rsconnect::writeManifest()
```

## Keeping both files in sync

When you add or update packages, update both files before pushing to GitHub:

```r
renv::snapshot()            # update renv.lock for local development
rsconnect::writeManifest()  # update manifest.json for Posit Connect deployment
```

## Summary

| | `renv.lock` | `manifest.json` |
|---|---|---|
| **Used by** | `renv` (local) | Posit Connect (server) |
| **Purpose** | Reproduce local dev environment | Deploy app to Posit Connect |
| **Updated by** | `renv::snapshot()` | `rsconnect::writeManifest()` |
| **Audience** | Developers | Posit Connect infrastructure |
